Question-1 : Write the differences between map, filter, for Each, find.

Answer: .map()

.map() executes the same code on every element in an array and returns a new array with the updated elements. map() maps an element of an array, performs an operation with it and returns the result in an array.

Let's see an example how map() warks. Let's say we have a list of numbers in an array and we want to square each of the array elements.


const numbers = [1, 2, 3, 4, 5]

const squares = numbers.map( x => x*x)

The output of this function will be

square = [1, 4, 9, 16, 25]

.forEach()

.forEach() calls a function for every element in an array and it does not returns anything. .forEach() calls a function for each element, operates on those elements and displays the final elements. It returns only single element.

Let's see an example how map() warks. Let's say we have a list of numbers in an array and we want to square each of the array elements.


const numbers = [1, 2, 3, 4, 5]

const squares = numbers.forEach( x => x*x)

The output of this function will be

square = 1, 4, 9, 16, 25

.filter()

The filter() method creates a new array with all those elements that fulfil the conditions that are given inside the filter(). It does not execute the function for empty elements. The original array remains the same. It does not change the original array/object.

let's see an example how filter() works. Let's say we have a list of numbers in an array and we want those values only who are greater than 10.


const numbers = [1, 5, 10, 15, 3, 7, 20, 32, 1]

const newNumbers = numbers.filter( x => x > 10)

The output of this function will be

newNumbers = [15, 20, 32]

.find()

The find() method the first element that fulfil the conditions that are given inside the find(). It returns a single element. It does not execute the function for empty elements. The original array remains the same. It does not change the original array/object.

let's see an example how find() works. Let's say we have a list of numbers in an array and we want those values only who are greater than 10.


const numbers = [1, 5, 10, 15, 3, 7, 20, 32, 1]

const newNumber = numbers.find( x => x > 10)

The output of this function will be

newNumber = 15

Question-1 : What is Event loop? How it works?

Answer:

The event loop is a constantly running process that monitors both the callback queue and the call stack. It manages asynchronous operations in a program. It is the central component of many event-driven architectures and is commonly used in web browsers, server-side frameworks, and other applications that require non-blocking I/O.

The event loop works by continuously monitoring a queue of events or tasks, and executing them one by one as they are added to the queue. When an asynchronous operation, such as a network request or a file read, is initiated, the event loop adds a corresponding event or task to the queue and continues to process other events in the meantime. Once the asynchronous operation is complete, the event loop retrieves the corresponding event from the queue and executes any callbacks or handlers associated with that event.

Various types of events, including timers, user input, and system events are also handled by the event loop. It ensures that events are processed in a timely and efficient manner, and that the program remains responsive even when multiple events are occurring simultaneously.

Overall, the event loop is a key component of many modern programming environments, that provides an efficient and scalable mechanism for managing asynchronous operations and enabling the development of highly responsive and efficient software.
